# CHANGELOG

## Unreleased

FEATURE: Git (`repo/`) subsystem — thin `git`-CLI wrappers in two shapes: a path-based read-only checkout from otis (`EnsureMirror`, `FetchBranch` with an explicit refspec so refs never go stale, `ResolveBranchSHA`, and ephemeral detached `CreateWorktree`/`RemoveWorktree`/`PruneWorktrees`/`CaptureHEAD`) and a `Repo` working-tree handle from sexton (`Status`/`StageAll`/`Commit`/`Pull --rebase`/`Push`/`Diff*` with sentinel errors like `ErrNothingToCommit`/`ErrConflict`/`ErrNoRemote`). The package is unopinionated about paths; SSH handling is unified (`IdentitiesOnly` + `StrictHostKeyChecking=accept-new`, shell-quoted key). sexton's branded commit-message helper and otis's state-layout coupling are dropped.

FEATURE: Mattermost + command subsystems — a `mattermost.Client` (REST `PostMessage`, WebSocket listen with reconnect, @mention/trigger-word command extraction, allowed-user filtering, bot-identity resolution), lifted from sexton, and a transport-agnostic `command.Registry` (`Register`/`Dispatch`/`Help`). The client routes a matched message to a `Responder func(ctx, command) string` and posts a non-empty reply, so wiring a registry to chat is one line (`mc.Start(reg.Dispatch)`) and the same registry can drive a CLI. sexton's domain handlers (status/sync/snooze/resume), formatters, and the alerter are dropped; trigger words no longer default to a tool name (empty means mention-only).

FEATURE: Initial reviewer subsystem — a backend-agnostic `Reviewer` interface with `codex`, `claude`, `pi`, and `dummy` adapters (lifted from mercurius's proven implementations; the codex adapter adds otis's per-reviewer `env` merge and an optional `config.toml`), schema-agnostic output validation, and a `GuardEnvelope` check that rejects the schema construct that broke otis's codex reviewer — nested objects or arrays inside array items. Ships two reference schemas: mercurius's verdict/concerns shape and a new code-review findings shape, confirmed live against codex and claude.

CHANGE: Stood up the project's agent-facing scaffolding — `AGENTS.md` pointing back at the grimoire substrate, the `docs/current` + `docs/future` + `docs/journal` structure, the reviewer-output contract, and this changelog.
